# 문제

연결 리스트(Linked List)는 데이터를 연속되지 않은 메모리 공간에 저장하는 기본적인 선형 자료구조입니다. 
연결 리스트의 흔한 연산으로는 삽입, 삭제, 탐색이 있지만, 이 문제에서는 **슬라이드(Slide)** 라는 특수한 연산을 다룹니다.

초기에 $1$부터 $N$까지의 정수가 순서대로 연결된 리스트가 있습니다. ($1 \rightarrow 2 \rightarrow 3 \rightarrow \cdots \rightarrow N$)
슬라이드 연산 `slide(a, b)`는 정수 $a$를 현재 위치에서 빼내어, 정수 $b$의 **바로 오른쪽(immediate right)** 위치로 이동시키는 연산입니다.

예를 들어 $N = 5$일 때 초기 상태는 $1 \rightarrow 2 \rightarrow 3 \rightarrow 4 \rightarrow 5$ 입니다.
- `slide(4, 1)`을 수행하면 $4$가 $1$의 오른쪽으로 이동하여 리스트는 $1 \rightarrow 4 \rightarrow 2 \rightarrow 3 \rightarrow 5$가 됩니다. 이때 $4$는 왼쪽으로 2칸 이동했으므로 이동 거리는 **-2**입니다.
- 이어서 `slide(1, 5)`를 수행하면 $1$이 $5$의 오른쪽으로 이동하여 $4 \rightarrow 2 \rightarrow 3 \rightarrow 5 \rightarrow 1$이 됩니다. 이때 $1$은 오른쪽으로 4칸 이동했으므로 이동 거리는 **4**입니다.
- 이어서 `slide(3, 2)`를 수행하면 $3$을 $2$의 오른쪽으로 이동시킵니다. 이미 $3$은 $2$의 오른쪽에 있으므로 리스트는 변하지 않으며 이동 거리는 **0**입니다.

$N$개의 정수로 이루어진 연결 리스트와 $Q$개의 슬라이드 연산이 주어질 때, 각 연산마다 정수 $a$가 이동한 거리를 구하고, 모든 연산이 끝난 후의 최종 연결 리스트 상태를 반환하세요.

## 함수 설명

### 매개 변수
- `n` : 연결 리스트에 포함된 정수의 개수 ($1 \le n \le 100,000$)
- `q` : 슬라이드 연산의 횟수 ($1 \le q \le 100,000$)
- `queries` : 슬라이드 연산의 목록. 각 연산은 `{a,b}` 형태의 배열이다. ($1 \le a,b \le n, a \neq b$)

### 반환값

크기가 $Q + N$인 1차원 숫자 배열을 반환해야 한다.
- 배열의 첫 $Q$개의 원소는 각 쿼리당 정수 $a$가 이동한 거리여야한다. (왼쪽 이동은 음수, 오른쪽 이동은 양수, 이동하지 않으면 0)
- 배열의 나머지 $N$개의 원소는 모든 연산이 종료된 후의 최종 연결 리스트의 상태이여야 한다.

## 제한
- $1 \le n, q \le 100,000$

- 서브 태스크 1 (10점) : $n,q \le 1,000$
- 서브 태스크 2 (15점) : $n \le 100,000, q \le 1,000$
- 서브 태스크 3 (20점) : 모든 연산에서의 정수 $a$의 실제 이동 거리의 절댓값은 100을 넘지 않는다.
- 서브 태스크 4 (25점) : 주어지는 $a,b$는 무작위로 생성된다.
- 서브 태스크 5 (30점) : 전체 제한
